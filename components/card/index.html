component/card Responsive Card

Component Description

This card component shows another strength of CSS Grid. Depending on the size of the viewport, the display of the component is changed without attacking the markup. This only works with the definition of grid-template-areas.

The card component gets a 3 column grid and is defined with the following areas: "photo", "header" and "description". To display the image left and header/description right for a big viewport, the card container must display the areas as follows: grid-template-areas: "photo header header" "photo description description";. 1 column and 2 rows for the image, 2 columns and 2 rows for the header and the description.

This behaviour can also be easily simulated with display: flex or float. But what to do if on mobile phones or tablets the elements should be positioned differently? Without JavaScript this was not possible.

grid-template-areas makes this possible. You redefine the areas in the desired viewport size. In this case, with a viewport size of 768px, the areas are displayed as follows: grid-template-areas: "header header header" "photo photo photo" "description" "description" "description" ;.

This is a simple and elegant solution to avoid manipulating the HTML markup with JavaScript.

HTML Markup order:

Image, Header, Description
Changed order of elements without changing the HTML markup with grid-template-areas to:
Header, Image, Description
