component/layout wrapper Layout Wrapper without creating an extra class

Component Description

Creating a wrapper component is essential for a website, because all the elements a website should represent shouldn't stick to the ends of a viewport. To give each element a margin or padding makes little sense and is prone to errors.

In this case there are two possibilities:
The first one is to create a wrapper class with the desired width and place this class in every HTML container element that should be aligned inside the wrapper.

The second one is working with CSS Grid. You define the grid-template-columns in the desired number of columns once. For a web page wrapper there are in this case 3 columns: grid-template-columns: 1fr [setAnyName] minmax(auto, 960px) 1fr; The first and last column take the available space and the named grid area [setAnyName] defines the wrapper with 960px width. Finally, you pass the desired HTML elements/classes in the stylesheet grid-column-start: setAnyName to align it in the wrapper.

I have to support:

Code

  • CSS
  • HTML
<div class="wrapper"> <div class="header__content">Header</div> <div class="main__content">Main Content</div> <div class="footer__content">Footer</div> </div>
